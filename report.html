Casino Analytics — отчёт о сервисе

ETL-пайплайн для аналитики игровой платформы: исходные CSV загружаются в PostgreSQL, трансформируются через dbt в витрину monthly_summary, результат показывается в Metabase и в виде Python-графиков. Всё поднимается одной командой в Docker.

Период данных: январь 2023 — февраль 2024 Сформировано: 2026-06-20

Архитектура

Оркестрация — Airflow на CeleryExecutor (scheduler, webserver, worker + Redis как брокер). DAG etl_monthly_summary прогоняет четыре шага по цепочке:

Источник

CSV в ./data — 7 таблиц (игроки, транзакции, курсы, справочники)

Airflow DAG

  1. load_csv → raw
  2. dbt run
  3. dbt test
  4. build_charts

PostgreSQL

raw.* → staging → витрина analytics.monthly_summary (VIEW)

Потребители

Metabase-дашборд и PNG/HTML-графики в ./reports

Все образы мультиарх (amd64 / arm64), поэтому стек одинаково поднимается на Windows и на Mac с Apple Silicon без эмуляции.

Как поднять

# из корня проекта
docker compose up -d --build

# дождаться, пока сервисы станут healthy
docker compose ps

# запустить ETL сразу, не дожидаясь расписания (@monthly)
docker compose exec airflow-scheduler airflow dags trigger etl_monthly_summary

Файл .env создавать не обязательно — дефолты зашиты в docker-compose.yml.

Качество данных

Шаг dbt test в DAG проверяет: not_null / unique на первичных ключах, ссылочную целостность (relationships), допустимые значения валют / стран / типов регистрации, отсутствие отрицательных сумм в витрине и уникальность зерна (month, country). Прогон проходит полностью.
